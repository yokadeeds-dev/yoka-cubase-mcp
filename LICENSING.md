# Lizenzierung

`yoka-cubase-mcp` (der freie Kern) wird unter einem **Dual-Lizenz-Modell** angeboten.

## 1. Open Source — AGPL-3.0

Standardmäßig steht dieser Code unter der **GNU Affero General Public License v3.0**
(voller Text in [`LICENSE`](LICENSE)). Du darfst ihn frei nutzen, studieren, modifizieren
und weitergeben — unter diesen Bedingungen:

- **Copyleft:** Abgeleitete Werke müssen ebenfalls unter AGPL-3.0 stehen.
- **Netzwerk-Klausel (§13):** Wenn du eine *modifizierte* Version als Netzwerk-Dienst
  betreibst (z. B. diesen MCP-Server für Dritte zugänglich machst), musst du den
  vollständigen Quellcode deiner Version den Nutzern zugänglich machen.
- Lizenz- und Copyright-Hinweise müssen erhalten bleiben.

Das ist die richtige Wahl für freie/offene Nutzung, Forschung, private Studios und
jedes Projekt, das selbst offen bleibt.

## 2. Kommerzielle Lizenz

Wenn du den Code in ein **proprietäres oder kommerzielles Produkt** einbetten willst,
**ohne** die AGPL-Pflichten (Copyleft, Quellcode-Offenlegung, Netzwerk-Klausel) zu
erfüllen, benötigst du eine **kommerzielle Lizenz**. Sie hebt die AGPL-Verpflichtungen
für dein Produkt auf — gegen eine Gebühr bzw. Vereinbarung.

**Anfragen:** per E-Mail an **yoka@provolution.org** oder über ein
[GitHub-Issue](https://github.com/yokadeeds-dev/yoka-cubase-mcp/issues).

## Warum dual?

So bleibt das Tool für die Community frei und offen — während kommerzielle Anbieter,
die es schließen oder weiterverkaufen wollen, einen fairen Beitrag leisten. Der volle
Funktionsumfang inkl. Mixing-/Mastering-Wissen (Nicker) ist Teil dieses Repos — es gibt
kein separates Premium-Paket.

## Beiträge

Externe Beiträge erfordern ein Contributor License Agreement — siehe
[`CONTRIBUTING.md`](CONTRIBUTING.md). Ohne CLA kann Fremd-Code nicht in den
kommerziellen Lizenz-Arm aufgenommen werden.
